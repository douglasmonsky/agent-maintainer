Below is an end-to-end implementation plan for refactoring Agent Maintainer into cleaner internal packages while preserving the single PyPI distribution initially. The current packaging setup already supports this shape because setuptools auto-discovers packages under `src`.

The intended end state is:

```text
src/
  agent_maintainer/        # Orchestrator: CLI, policies, verify profiles, hooks integration
  agent_repair_facts/      # Normalized repair facts from heterogeneous tool output
  agent_context/           # Budgets, context packs, safe expansion, compression
  agent_run_artifacts/     # Versioned verifier artifact schemas and rendering
  agent_client_hooks/      # Client-specific hook config/templates/adapters
  docsync/                 # Existing docs evidence and claim-freshness package
  archguard/               # Existing architecture/config validation package
```

Keep the public distribution as `agent-maintainer` for now. The internal package split gives you clean seams without creating release/version friction.

# Phase 1: Define package ownership and dependency direction

The key rule:

```text
agent_maintainer may import all internal packages.

agent_repair_facts must not import agent_maintainer.
agent_context must not import agent_maintainer.
agent_run_artifacts must not import agent_maintainer.
agent_client_hooks should not import agent_maintainer except through explicit adapter shims.
docsync must not import agent_maintainer or archguard.
```

Target dependency graph:

```text
docsync
  depends on: stdlib only and package-declared dependencies; it must remain extractable

agent_repair_facts
  depends on: stdlib only, defusedxml if XML parsing requires it

agent_context
  depends on: agent_repair_facts

agent_run_artifacts
  depends on: agent_context, agent_repair_facts

agent_client_hooks
  depends on: stdlib only for templates/config generation

agent_maintainer
  depends on: all of the above
```

This lets Agent Maintainer stay the product orchestrator while the smaller packages become reusable primitives.

Add a short architecture note before large moves:

```text
docs/architecture/decisions/YYYY-MM-DD-internal-package-boundaries.md
```

Include:

```text
Decision:
Split reusable primitives into internal packages under src while keeping one distribution.

Boundaries:
- agent_repair_facts owns diagnostic normalization.
- agent_context owns bounded context construction.
- agent_run_artifacts owns manifest/artifact schemas.
- agent_client_hooks owns client-specific hook config generation.
- docs_evidence owns code/docs evidence indexing.
- agent_maintainer owns policy, CLI orchestration, profiles, and product workflow.

Tach:
Each new top-level package gets an explicit root module entry or a tach.domain.toml contract.
```

This matters because your repo’s architecture policy explicitly says Tach policy changes need an ADR under `docs/architecture/decisions/`.

## 4.2 Define the stable model

Create `models.py`:

```python
from __future__ import annotations

from dataclasses import dataclass
from pathlib import Path
from typing import Any


@dataclass(frozen=True)
class RepairFact:
    """One normalized, agent-actionable repair fact."""

    check: str
    message: str
    severity: str = "error"
    path: str | None = None
    line: int | None = None
    column: int | None = None
    end_line: int | None = None
    end_column: int | None = None
    symbol: str | None = None
    rule: str | None = None
    category: str | None = None
    source: str | None = None
    raw_excerpt: str | None = None
    confidence: str = "high"

    def to_payload(self) -> dict[str, object]:
        """Return backward-compatible context-pack payload."""
        return {
            "check": self.check,
            "path": self.path,
            "line": self.line,
            "column": self.column,
            "end_line": self.end_line,
            "end_column": self.end_column,
            "symbol": self.symbol,
            "rule": self.rule,
            "category": self.category,
            "message": self.message,
            "severity": self.severity,
            "source": self.source,
            "raw_excerpt": self.raw_excerpt,
            "confidence": self.confidence,
        }


@dataclass(frozen=True)
class ParserInput:
    """Inputs available to one repair-fact parser."""

    check: str
    path: Path
    source: str = ""
```

Keep the payload keys aligned with existing context pack facts. The point of this phase is to move ownership, not change output shape.

```python
from __future__ import annotations

from agent_repair_facts.models import RepairFact


def fact_payload(payload: dict[str, object]) -> dict[str, object]:
    """Return stable repair-fact payload with all expected keys."""
    return RepairFact(
        check=str(payload.get("check", "unknown")),
        message=str(payload.get("message", "")),
        severity=str(payload.get("severity", "error")),
        path=optional_str(payload.get("path")),
        line=optional_int(payload.get("line")),
        column=optional_int(payload.get("column")),
        end_line=optional_int(payload.get("end_line")),
        end_column=optional_int(payload.get("end_column")),
        symbol=optional_str(payload.get("symbol")),
        rule=optional_str(payload.get("rule")),
        category=optional_str(payload.get("category")),
        source=optional_str(payload.get("source")),
        raw_excerpt=optional_str(payload.get("raw_excerpt")),
        confidence=str(payload.get("confidence", "high")),
    ).to_payload()


def optional_str(value: object) -> str | None:
    return value if isinstance(value, str) and value else None


def optional_int(value: object) -> int | None:
    return value if isinstance(value, int) else None
```

# Phase 11: Prevent major regressions with architectural tests

Add explicit tests that enforce the new package boundaries.

Create:

```text
tests/architecture/test_internal_package_boundaries.py
```

Test no upward package imports `agent_maintainer`:

```python
from __future__ import annotations

import ast
from pathlib import Path

UPWARD_PACKAGES = (
    "src/agent_repair_facts",
    "src/agent_context",
    "src/agent_run_artifacts",
    "src/agent_client_hooks",
    "src/docs_evidence",
)


def imported_roots(path: Path) -> set[str]:
    tree = ast.parse(path.read_text(encoding="utf-8"))
    roots: set[str] = set()
    for node in ast.walk(tree):
        if isinstance(node, ast.Import):
            for alias in node.names:
                roots.add(alias.name.split(".", maxsplit=1)[0])
        elif isinstance(node, ast.ImportFrom) and node.module:
            roots.add(node.module.split(".", maxsplit=1)[0])
    return roots


def test_upward_packages_do_not_import_agent_maintainer() -> None:
    offenders = []
    for root in UPWARD_PACKAGES:
        for path in Path(root).rglob("*.py"):
            if path.name == "__init__.py":
                continue
            if "agent_maintainer" in imported_roots(path):
                offenders.append(path.as_posix())
    assert offenders == []
```

Add allowed dependency tests:

```python
ALLOWED_IMPORTS = {
```

```text
- Add changelog note.
- Run full verification.
```

For a beta package, keeping shims is safer. It prevents accidental import regressions for users who started relying on internals.

# Phase 17: Specific regression risks and how to catch them

## Risk: new packages are not scanned by Agent Maintainer

Cause: `source_roots`, `package_paths`, `coverage_source`, `file_length_paths`, `structure_paths`, or `vulture_paths` still omit new packages.

Detection:

```bash
python3 -m agent_maintainer guidance
grep -n "Source roots" AGENTS.agent-maintainer.md
python3 -m agent_maintainer verify --profile precommit
```

The generated guidance should list all new source roots because it renders them from config.

## Risk: coverage drops unexpectedly

Cause: new packages are included in `coverage_source`, but tests do not cover moved logic.

Detection:

```bash
pytest --cov=src/agent_repair_facts --cov=src/agent_context --cov=src/agent_run_artifacts
python3 -m agent_maintainer verify --profile precommit
```

Fix by moving existing tests to import new package paths and adding direct tests for new APIs.

## Risk: Tach fails due unowned modules

Cause: new package files exist but are not declared in root or domain Tach config.

Detection:

```bash
python3 -m archguard.tach_config
tach check --exact
```

Archguard has tests requiring explicit source module ownership and dependency contracts.

## Risk: Tach passes by using broad path buckets

Cause: someone adds `paths = [...]` with too many unrelated modules.

Detection:

```bash
python3 -m archguard.tach_config
```

Archguard rejects broad groups larger than the configured maximum in both root and domain Tach files.

## Risk: context packs change shape

Cause: moved code changes payload keys or nesting.

Detection:

```bash
python3 -m agent_maintainer context pack --format json > /tmp/pack.json
python3 - <<'PY'
import json
payload = json.load(open("/tmp/pack.json"))
required = {
    "exact_repair_facts",
    "supporting_context",
    "compression",
    "untrusted_content_labels",
    "ratchet_state",
    "top_targets",
    "selected_file_outlines",
    "selected_logs",
    "omitted_counts",
    "expansion_commands",
    "outputs",
}
missing = required - set(payload)
assert not missing, missing
PY
```

The current payload includes those keys.

## Risk: hook output becomes noisy

Cause: hook failure context prints full verifier output or full context pack.

Detection:

```bash
python3 .claude/hooks/post_tool_use.py < /tmp/empty-hook-payload.json > /tmp/hook.out || true
wc -c /tmp/hook.out
cat /tmp/hook.out
```

Expected:

```text
- bounded JSON response
- compact reason
- context pack pointer
- top findings
- expansion commands
```

Hook docs say hook output should include the pack path, up to three exact repair facts, and safe expansion commands within `context_hook_budget_chars`.

## Risk: old imports break

Cause: moved files without shims.

Detection:

```bash
python3 - <<'PY'
import agent_maintainer.context.budget
import agent_maintainer.context.failures
import agent_maintainer.context.reading.logs
import agent_maintainer.context.pack.builder
import agent_maintainer.context.pack.fact_parsers
import agent_maintainer.verify.artifact_manifest
import agent_maintainer.verify.pr_summary
import agent_maintainer.hooks.templates
print("compat ok")
PY
```

## Risk: wheel misses packages

Cause: package discovery or package data issue.

Detection:

```bash
python3 -m build
python3 -m pip install --force-reinstall dist/*.whl
python3 - <<'PY'
import pkgutil
for name in [
    "agent_repair_facts",
    "agent_context",
    "agent_run_artifacts",
    "agent_client_hooks",
    "docs_evidence",
]:
    assert pkgutil.find_loader(name), name
print("packages found")
PY
```

# Phase 18: Final acceptance criteria

The refactor is complete when all of these are true:

```text
1. New internal packages exist:
   - agent_repair_facts
   - agent_context
   - agent_run_artifacts
   - agent_client_hooks
   - docsync remains the extractable docs/evidence package

2. Upward packages do not import agent_maintainer.

3. Agent Maintainer uses the new packages as an orchestrator.

4. Old imports either still work through shims or have been intentionally removed with tests/docs updated.

5. Tach root and domain contracts explicitly own all new modules.

6. pyproject Agent Maintainer config includes new packages in:
   - source_roots
   - package_paths
   - coverage_source
   - file_length_paths
   - structure_paths
   - vulture_paths
   - semgrep_args
   - cohesive_change_override_paths

7. Generated guidance is current.

8. Context pack JSON shape is stable.

9. Hook output remains bounded.

10. Run artifacts remain stable:
    - manifest.json
    - LAST_FAILURE.md
    - pr-summary.md
    - run-scoped snapshots

11. Full verification passes:
    - pytest
    - pyright
    - ruff
    - tach check --exact
    - agent_maintainer doctor
    - precommit profile
    - full profile
    - ci profile
    - security profile
    - manual profile
```

The core implementation principle is:

```text
Move ownership first.
Preserve behavior through shims.
Add direct tests for the new package.
Update Tach/config/guidance immediately after each move.
Only then remove duplication.
```

This keeps the refactor reviewable while avoiding the two big failure modes: architecture drift that only appears at the end, and subtle behavior changes hidden inside file moves.
