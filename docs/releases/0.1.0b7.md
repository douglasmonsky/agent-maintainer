## Candidate Highlights

- Owner verification policy and worktree doctor diagnostics now agree on
  configuration, managed hooks, and safe artifact health.
- Pull-request and release verification fan out independent verifier-owned
  groups and aggregate exact-state evidence before protected results pass.
- Local commit hooks run fast staged policy plus affected tests; pre-push runs
  the complete profile against the exact outgoing range from a clean checkout.
- Workflow dependency and external-tool caches are bound to reviewed versions
  and hashes, while release publishing intentionally remains cache-free.
- Public commands carry explicit core, optional, or experimental stability
  labels.
- Repository-unused run-artifact compatibility facades are removed in favor of
  the canonical `agent_run_artifacts` package.
