## Release Highlights

- Introduces a disabled-by-default experimental C/C++ (CMake) provider
  foundation with explicit beta configuration and conservative file
  classification.
- Reports advisory C/C++ suppression evidence and static doctor diagnostics;
  neither surface adds blocking policy.
- Rejects unsafe CMake roots, symlinked repository wrappers, and inexact
  suppression markers instead of treating ambiguous input as trusted evidence.
- Exposes the optional `cpp` field through generated capability metadata and
  the configuration reference.

Phase 187 never executes configured C/C++ commands. Compiler, build, test,
coverage, formatting, and analysis execution remain Phase 188 work. The
compatible optional configuration-member addition keeps contract revision 1.

There are no breaking changes to existing C/C++, Python, TypeScript, Java,
CLI, or verifier contracts. The optional C/C++ configuration member is
additive, remains disabled by default, and keeps contract revision 1.
