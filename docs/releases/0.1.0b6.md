# Agent Maintainer 0.1.0b6 Candidate Notes

## Candidate State

- Status: `unpublished`
- Target version: `0.1.0b6`
- Latest published version: [`0.1.0b5`](0.1.0b5.md)
- Upgrade guidance: [`upgrading-to-0.1.0b6.md`](../upgrading-to-0.1.0b6.md)

These notes describe candidate intent only. They do not claim package-index
publication, a GitHub release, immutable build identifiers, or artifact
digests. The record must be replaced with exact evidence if publication
succeeds.

## Candidate Highlights

- Repository-controlled MCP and DocSync filesystem operations are bounded to
  approved roots and reject unsafe files and destinations.
- Hook installation, initialization, update, and uninstall share one managed
  inventory and a previewable transactional mutation contract.
- Invalid resolved configuration fails before command behavior begins.
- Detached verifiers and durable waits retain explicit state across terminal
  closure, with targeted resume and rewake paths.
- Release workflows require exact-commit verification, immutable actions, and
  digest-bound wheel and source-distribution transfers.
- Public beta governance provides private security reporting, explicit support
  boundaries, contribution and conduct policies, ownership routing, and
  structured issue intake.
- Runtime events, attention and recall ledgers, efficacy assessment, and repair
  facts provide more precise local feedback.
- TypeScript/React and task-broker work remains advisory or experimental; the
  stable blocking contract remains Python-first.

## Candidate Evaluation

Use the [upgrade guide](../upgrading-to-0.1.0b6.md) in a disposable or clean
test checkout. Review the complete [Unreleased changelog](../../CHANGELOG.md)
for the user-impact reconstruction since `0.1.0b5`.

The candidate is not ready to become the published pointer until every item in
the [critical stabilization definition of done](../roadmap/critical-stabilization.md#stabilization-definition-of-done)
passes for one exact commit.
