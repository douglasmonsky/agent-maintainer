## Release Highlights

- `agent-maintainer verify-plan` maps a Git diff to affected Python packages,
  TypeScript workspaces, Gradle modules, repository fallback units, named
  checks, canonical verifier profiles, review categories, and evidence.
- Versioned `.agent-maintainer/path-risk.toml` policy lets repositories declare
  sensitive paths and the exact evidence or review those changes require.
- Adds, modifications, copies, renames, deletions, and type changes are
  preserved. Rename and copy sources can trigger policy, while only current or
  destination paths can satisfy changed-path evidence.
- Strict repository-relative paths, bounded segment-aware globs, confined
  workspace discovery, and fail-closed policy validation keep planning safe and
  deterministic.
- Human output and `schema_version = 1` JSON are byte-stable for identical
  facts. Enforcement distinguishes missing evidence from invalid input with
  exact exit statuses.
- The optional `verification-plan-policy` gate integrates repository policy
  into existing profiles without executing a reduced test set or suppressing
  any configured verifier gate.

There are no breaking changes to existing CLI, configuration, or verifier
execution contracts. Verification planning is additive; repositories opt into
policy by adding `.agent-maintainer/path-risk.toml`.
