## Candidate Highlights

- The experimental Java/Gradle provider requires a checked-in wrapper and uses
  two grouped wrapper calls for static analysis and tests.
- Reviewed Groovy and Kotlin DSL setup covers Spotless, SpotBugs, Checkstyle,
  PMD, JaCoCo, file/finding baselines, and upward-only coverage thresholds.
- Structured report parsing is bounded and safe, with exact repair facts and
  truthful multi-project topology instead of synthetic aggregate percentages.
- Live Gradle validation is cached and configured for Linux and Windows, while
  provider promotion still requires hosted Windows proof and production-scale
  repository calibration.
- Cohesive-change accounting now includes copied and renamed destinations.
