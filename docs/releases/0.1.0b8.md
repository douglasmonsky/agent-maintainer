## Release Highlights

- The experimental Java/Gradle provider requires a checked-in wrapper and uses
  two grouped wrapper calls for static analysis and tests.
- Reviewed Groovy and Kotlin DSL setup covers Spotless, SpotBugs, Checkstyle,
  PMD, JaCoCo, file/finding baselines, and upward-only coverage thresholds.
- Structured report parsing is bounded and safe, with exact repair facts and
  truthful multi-project topology instead of synthetic aggregate percentages.
- Live Gradle validation is cached and configured for Linux and Windows with
  platform-native ratchet probe line endings, while provider promotion still
  requires hosted Windows proof and production-scale repository calibration.
- Cohesive-change accounting now includes copied and renamed destinations.
- TypeScript/JavaScript now exposes advisory package-manager and workspace
  evidence, configured Knip unused-code/dependency facts, grouped OSV
  vulnerability facts, and dependency-cruiser architecture findings.
- Existing LCOV artifacts can be compared with Git changed lines through an
  explicit advisory report with weighted executable-line math, separate
  missing-file evidence, strict repository confinement, and bounded output.
- Public npm and pnpm-workspace projections exercise the new TypeScript facts,
  while the provider remains experimental and does not infer or install tools,
  invent commands or rules, or enforce a coverage threshold.
- Base installs include the Java report parser dependency, and packaged source
  syntax remains compatible with the declared Python 3.11 through 3.14 range.
