## Release Highlights

- Opt-in semantic compatibility ratchets cover repository configuration,
  authored CLI manifests, nominated Python APIs, and structural JSON Schema.
- Deterministic diff, enforcement, and explicit snapshot commands connect exact
  semantic changes to revision, package-version, migration, and review-decision
  obligations.
- The optional verifier gate uses index-authoritative staged inputs at
  pre-commit and dogfoods five Agent Maintainer contracts.
- Fail-closed handling covers unsafe paths and Git refs, non-regular or
  ambiguous staged entries, deep baseline data, and base refs that predate
  ratchet adoption.

There are no breaking changes to existing CLI, configuration, or verifier
execution contracts. Semantic contract ratchets are additive and opt-in.

## Adoption

Both package indexes were smoke-tested from clean Python 3.14 environments.
The smokes installed the `core` extra, ran `agent-maintainer --help`,
`archguard --help`, and `docsync --help`, and initialized a synthetic repository
with `agent-maintainer init --track core`. Use the
[upgrade guide](../upgrading-to-0.1.0b10.md) for normal package-index adoption,
behavior review, and rollback guidance.
